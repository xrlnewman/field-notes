# StoreFlow 门店经营预约 SaaS 设计

## 产品定位

StoreFlow 是面向美容、美甲、洗护、健身私教、摄影和汽车服务门店的免费开源经营底座。它把“顾客选门店—选择项目—锁定时段—到店核销—会员复购”串成一条可追踪链路，重点解决小门店排班混乱、预约冲突和会员复购缺少数据的问题。

## 与现有作品的区分

- HomeFlow 解决上门服务的派单与履约。
- 邻里社区解决团购、跑腿和社区关系。
- 多商户商城解决商品交易。
- StoreFlow 只做“到店预约 + 门店排班 + 会员经营”，不复用邻里项目作为下一款产品。

## MVP 范围

### 顾客小程序

- 门店列表：按距离、服务类型和评分浏览门店。
- 门店详情：展示门店信息、项目卡片、价格、时长和可预约日期。
- 预约流程：选择项目、员工、日期和时段，提交预约并获得预约号。
- 我的预约：按待到店、已完成、已取消查看预约，支持取消和再次预约。
- 会员卡：展示余额、积分和到期时间；本期只读展示，不接支付。

### 门店后台

- 运营总览：今日预约、到店率、营业额、会员复购率。
- 门店与员工：门店资料、员工技能、班次和请假。
- 服务项目：价格、时长、缓冲时间、上下架状态。
- 预约调度：按日历和列表查看预约，改期、确认到店、完成核销。
- 会员管理：会员详情、积分、消费记录和标签。

### 后端 API

- Go 1.25 + Gin，MySQL 8.4 持久化门店、员工、服务、时段、预约、会员和审计记录。
- Redis 8 负责预约时段短锁、幂等键和热点门店缓存。
- JWT + RBAC：顾客、员工、店长、平台管理员四类角色。
- 所有写接口使用统一响应 `{ code, message, data }`，列表统一分页。
- 预约写入必须使用业务唯一键和时段锁，重复提交返回同一预约结果。

## 视觉方向

StoreFlow 不沿用 HomeFlow 的深青蓝 + 橙色，而使用“靛蓝运营感 + 杏黄色行动色 + 薄荷成功色”：

- `#4F46E5` 靛蓝：品牌、导航、选中态和主要标题。
- `#F59E0B` 杏黄：预约、确认、价格和待处理提醒。
- `#10B981` 薄荷绿：到店、完成、可预约和成功提示。
- `#F7F7FC` 淡紫灰：页面画布；`#FFFFFF` 卡片；`#1F2937` 正文。
- 图表使用靛蓝、杏黄、青绿、玫红四组系列色，避免单色图表。

## 非目标

- 本期不接在线支付、退款、发票和第三方会员卡支付。
- 本期不做微信 OAuth 和短信验证码，使用演示登录和可替换认证接口。
- 不把 HomeFlow 或邻里仓库复制为 StoreFlow 的公开源码历史。

## 验收标准

1. 两个公开仓库可独立安装、测试和构建：`storeflow-miniapp` 与 `storeflow-admin`。
2. 后台仓库包含 `web` 与 `server`，Go 服务提供门店、项目、时段、预约和会员接口。
3. 小程序至少能完成“选择门店 → 选择项目 → 选择时段 → 创建预约 → 查看预约”。
4. 预约时段重复提交不会产生重复预约，Redis 不可用时错误明确并可回退到演示数据。
5. 博客项目页展示完整截图、双仓库 GitHub 链接、技术栈和免费开源说明。
