# GitHub 项目清理与博客个人品牌补全设计

## 目标

让 GitHub 和个人博客形成一致的个人品牌入口：GitHub 只保留并突出可浏览的网站项目源码；博客首页和关于页能在首屏明确看到许汝林的头像、身份、7 年经验、能力边界与联系入口。

## 已确认现状

- GitHub 账号 `xrlnewman` 当前有 18 个公开仓库，其中 10 个属于四个网站项目，8 个属于桌面或数据工具。
- GitHub 主页固定仓库中，只有 `field-notes` 是网站项目，其余固定项主要是工具仓库。
- GitHub 已有头像，但个人资料没有中文姓名、职业简介和博客地址。
- 博客已有姓名、年龄、经验和技术栈文字，但首页右侧只有统计数字，关于页也没有头像，个人身份不够完整。
- 博客仓库仍保留 8 个不再使用的工具项目截图。

## 方案选择

采用已批准的“彻底清理”方案：删除远程工具仓库，本地源码保留；GitHub 主页只固定四个网站项目的代表仓库；博客使用现有 GitHub 头像作为统一身份资产，并完善首页与关于页的个人档案。

未采用以下方案：

- 只取消固定：工具仍会出现在仓库列表中，不符合“GitHub 只要网站项目”。
- 改为私有：仍占用账号维护空间，也不能形成干净的公开项目结构。

## 博客信息架构与视觉设计

### 统一头像资产

- 使用当前 GitHub 头像作为真实来源，不虚构人物照片。
- 将头像下载为博客本地静态资源，避免页面依赖 GitHub 外链和网络状态。
- 头像在首页、关于页和社交分享信息中使用同一裁切与替代文本。

### 首页

保留现有星空主题和三种主题切换，不改变项目、文章、留言的主导航。

首屏改为左右双栏：

- 左侧保留核心价值主张、两句个人简介和“查看项目 / 了解我”操作。
- 右侧将单纯统计列表升级为个人身份卡，包含头像、姓名、角色、7 年经验、四个网站产品、永久免费开源承诺和主要技术标签。
- 身份卡保持半透明星空玻璃质感，并适配三种主题；不使用新的无关配色。

项目内容仍是首页主体，个人卡只负责建立信任，不抢项目截图的视觉层级。

### 关于页

首屏改为“人物档案 + 价值主张”结构：

- 左侧放头像、姓名、角色和可联系状态。
- 右侧放现有主标题、完整简介以及 GitHub / 留言板入口。
- 年龄、经验、全栈边界保留为数据条。
- 现有三块能力说明保留并优化排版，补充技术栈标签，使“会什么、能解决什么”一眼可读。

## 组件边界

- 新建可复用的个人身份组件，负责头像、姓名、角色、简介、能力标签和联系入口。
- 首页负责传入紧凑展示模式和数据统计。
- 关于页负责传入完整展示模式和详细文案。
- 个人信息仍集中在 `src/config/site.ts`，页面不重复硬编码姓名、年龄、经验和链接。
- 页面样式延续现有 CSS 变量、字体、圆角和响应式断点，不进行无关重构。

## 清理与数据流

1. `siteConfig` 提供个人资料、头像路径、技术标签和社交链接。
2. 首页与关于页读取同一份配置并渲染个人身份组件。
3. 删除博客 `public/images/projects/` 下 8 个未引用的工具截图，四个网站项目截图保持不变。
4. 先完成博客代码、测试和部署，再整理 GitHub 主页与删除远程工具仓库，避免 GitHub 展示出现短暂空白。

## 失败处理与安全边界

- 删除远程仓库前逐个核对本地源码目录；缺少本地副本时停止删除该仓库并报告。
- GitHub 操作逐项执行并在每项后核对结果，不批量猜测成功状态。
- 不删除十个网站项目仓库，不修改其源码和关联关系。
- 不修改评论/留言的 Giscus 站内交互方式。
- 不提交任何 GitHub 或 Cloudflare 令牌。
- 若 GitHub 认证或删除权限不足，博客改造可继续，但 GitHub 清理必须明确列出未完成项。

## 测试与验收标准

### 自动验证

- 为个人配置、头像资源、首页身份卡、关于页档案和工具截图清理添加契约测试。
- 运行 `npm test`、`npm run check`、`npm run test:build`。
- 运行 `codegraph sync`，确认索引更新。

### 浏览器验收

- 桌面与移动宽度下检查首页和关于页，无溢出、裁切、字体错位或不可读内容。
- 三种星空主题下头像、卡片、按钮和文字对比度正常。
- 首页项目、文章、关于、GitHub 和留言入口均可点击。
- 浏览器控制台无错误和警告。
- 线上部署后重新检查首页与关于页。

### GitHub 验收

- 公开仓库列表只剩十个网站项目仓库。
- 主页固定项只有四个网站项目代表仓库。
- 主页显示中文姓名、职业简介和博客地址。
- 博客项目链接均能到达保留的 GitHub 仓库。
