# 项目真实截图画廊设计

## 目标

把项目详情从“单张封面 + 文字说明”升级为可验证产品能力的真实案例展示。每个公开项目提供 4～6 张来自对应本地网站源码的页面截图，覆盖用户前台、核心功能、运营后台与移动端场景；首页和项目目录继续只使用一张代表性封面，保持浏览效率。

## 采用方案

采用真实截图画廊，不生成虚构界面，不把多张页面拼成难以阅读的海报，也不增加自动播放视频。

- 项目卡片：展示一张最有代表性、信息密度适中的横向封面。
- 项目详情：首屏下方展示“产品界面”画廊，包含主预览、缩略图、场景标题和说明。
- 点击查看：使用原生 `dialog` 打开大图，支持关闭按钮、Esc 关闭、上一张和下一张。
- 响应式：桌面截图保持宽屏比例；移动截图以独立手机画框展示，不拉伸、不裁掉主要内容。
- 三种星空主题共享同一套布局，仅沿用各主题已有颜色变量、边框和光晕。

## 内容模型

项目内容模型新增 `screenshots`：

```yaml
screenshots:
  - src: /images/projects/field-notes/home.png
    alt: 许汝林个人博客首页
    title: 个人首页
    caption: 星空主题下集中展示项目、文章与个人能力。
    viewport: desktop
    width: 1440
    height: 900
```

字段约束：

- `src`：站内绝对路径，必填。
- `alt`：描述页面内容，不使用“图片”或“截图”等空泛措辞，必填。
- `title`：画廊中的短标题，必填。
- `caption`：说明该页面展示的用户任务或产品能力，必填。
- `viewport`：`desktop` 或 `mobile`，必填。
- `width`、`height`：原图像素尺寸，均为正整数，用于避免页面加载时跳动，必填。

公开项目至少 4 张截图，最多 6 张。`cover` 仍然独立存在，用于列表卡片与社交预览，不从数组位置隐式推断。

## 页面结构

`ProjectLayout` 在项目封面之后、产品全景之前插入 `ProjectScreenshots`：

1. 左侧为当前图片标题、场景说明和 `01 / 05` 进度。
2. 右侧为主预览；桌面截图占满容器，移动截图在深色设备画框内居中。
3. 下方缩略图横向排列；当前项有明显边框和 `aria-current`。
4. 点击主预览或缩略图切换；“查看大图”打开原生对话框。

移动端改为单列，说明在上、主图在下，缩略图保持横向滚动；交互目标不小于 44px。

## 资源处理

- 复用关联项目仓库中的真实截图原文件，复制到 `public/images/projects/<project-id>/`。
- 文件名使用英文场景名，不保留测试序号或临时命名。
- 不改变原始长宽比；浏览器负责按 `viewport` 类型适配展示。
- 首张详情图可急切加载，其余图片懒加载；保留明确的宽高属性以减少页面跳动。
- 不公开令牌、账号、内部域名或真实用户隐私数据。

## 可访问性与失败处理

- 画廊无需 JavaScript 时仍按静态图片列表展示，内容可阅读。
- JavaScript 增强后，缩略图使用按钮，当前项通过 `aria-current` 表达。
- 对话框有可见标题与关闭按钮，Esc 可退出；上一张/下一张按钮有中文可访问名称。
- `prefers-reduced-motion` 下取消图片切换动画。
- 图片加载失败时保留标题和说明，避免整个区块消失。

## 验收标准

1. 四个公开项目每个包含 4～6 张真实页面截图。
2. 邻里社区卡片不再使用空数据后台页，所有卡片封面清晰且比例一致。
3. 详情页可通过缩略图切换图片，可点击查看大图，并可使用键盘关闭。
4. 桌面、平板、手机均无横向页面溢出，移动截图不被拉伸。
5. 三种主题下文字、边框和当前项状态清晰可见。
6. 无脚本场景仍能看到全部截图与说明。
7. 现有 Giscus 评论、仓库链接、项目筛选和星空交互保持不变。

## 非目标

- 不制作自动播放宣传视频。
- 不新增后台上传截图能力。
- 不把项目截图托管到第三方图床。
- 不改动关联项目本身的产品功能。
