# 三主题星空作品集设计规格

## 目标

把现有个人博客重做为以网站产品为核心的沉浸式星空作品集。访客应在首屏明确看到许汝林的身份、7 年经验、网站项目能力和“个人博客永久免费、零成本部署、完全开源”的承诺，并能在同一页面切换三种完整视觉主题。

## 信息架构

首页公开展示四个产品，不再展示桌面工具、开发工具或数据工具：

1. 许汝林个人博客；
2. 多商户 SaaS 商城；
3. 邻里社区服务平台；
4. 天舶重工多语言企业官网。

每张卡片代表一个完整产品。商城、邻里、天舶的前台、后台和服务端不是独立作品卡片，而是在项目详情页的“关联源码”区域作为同一产品的组成节点展示。

文章、评论、项目评论、全局留言板和搜索继续保留。主题系统覆盖所有公共页面，但首页与项目页拥有最完整的星空互动。

## 三种主题

### 深空观测站（默认）

- 标识：`observatory`；
- 主色：深海军蓝、冰蓝、靛青；
- 氛围：克制、清晰、适合长时间阅读；
- 星空行为：慢速漂移、鼠标附近的星点连线、轻微景深视差。

### 梦幻银河

- 标识：`nebula`；
- 主色：午夜紫、玫红、青紫；
- 氛围：柔和星云、流星和散射光；
- 星空行为：鼠标点击产生星光涟漪，移动时出现短暂星尘尾迹。

### 宇宙终端

- 标识：`terminal`；
- 主色：近黑、荧光绿、电子青；
- 氛围：轨道扫描、细网格、坐标与数据读数；
- 星空行为：鼠标移动更新坐标反馈，悬停项目时出现扫描边框。

## 主题切换

- 顶部导航提供三个带文字的分段按钮，桌面端显示完整名称，窄屏显示短名称；
- 切换不刷新页面，使用 280ms 的颜色与材质过渡；
- 选择保存在 `localStorage` 的 `cosmic-theme` 键；
- 没有有效保存值时使用 `observatory`；
- 页面脚本在首屏绘制前设置 `data-theme`，避免闪烁；
- 切换后派发 `cosmic-theme-change` 自定义事件，星空画布据此更新调色板和行为。

## 星空与鼠标交互

- 使用单个固定定位的 Canvas 2D 背景，不引入 WebGL 或大型动效依赖；
- 画布位于内容下方、页面底色上方，不拦截点击；
- 桌面端根据视口面积生成星点，数量限制在 70–180；移动端限制在 45–90；设备像素比最高按 2 处理；
- 鼠标移动控制近景星层的轻微视差，并只计算指针半径内的连线；
- 指针按下生成寿命有限的涟漪；银河主题增加星尘，终端主题显示坐标；
- 项目卡片根据指针位置产生不超过 2.5 度的倾斜和局部高光，离开后回正；
- 可视区进入动画使用 IntersectionObserver，一次性完成；
- `prefers-reduced-motion: reduce` 下停止循环动画、取消视差/倾斜，只保留静态星空和即时主题切换；
- 无精细指针的设备不显示自定义指针光晕，触摸只触发一次涟漪。

## 字体与排版

- 字体随仓库构建并由本站提供，不依赖远程字体服务；
- 中文正文与标题使用 Noto Sans SC，英文、数字和界面标签使用 Inter；
- 只使用标准字重 400、500、600、700；
- 中文标题不得使用负字间距，英文标签最多使用 `0.08em` 正字距；
- 正文桌面端为 17px、行高 1.75，移动端为 16px、行高 1.7；
- 首屏标题控制在桌面端 64–84px、移动端 42–56px，避免遮挡项目信息。

## 首页布局

1. 透明深色导航：品牌、页面入口、三主题切换和移动菜单；
2. 首屏：身份标签、主标题、能力简介、项目与文章入口、7 年经验/4 个网站产品/永久免费三个数据节点；
3. 免费开源声明：个人博客以独立发光标签突出“永久免费 · 零成本部署 · 完全开源”；
4. 精选产品：四张宽幅案例卡，桌面端两列，首个博客项目横跨整行；
5. 最近文章；
6. 页脚与留言入口。

视觉层次以大截图、标题和留白为主，不使用密集小胶囊、过小正文或三列工具卡片。

## 产品卡片与详情

- 卡片包含真实界面截图、产品类型、状态、简介、最多三个技术标签和关联仓库数量；
- 卡片主操作进入站内详情，次操作只在单仓库产品上直接打开源码；
- 详情页新增“产品全景”区域，按 `frontend`、`admin`、`backend`、`content` 角色展示关联仓库；
- 每个仓库节点包含名称、角色说明、技术摘要和 GitHub 链接；
- `repoUrl` 继续作为主仓库兼容字段，`repositories` 保存完整关联关系；
- 所有 GitHub 链接使用 `target="_blank"`、`rel="noopener noreferrer"`。

## 内容与文案

个人博客详情页必须明确：

- 网站访问与源码永久免费；
- 部署方案以 Cloudflare Pages、Functions、D1 的免费额度为基础；
- 文章、项目、站内评论和全局留言全部留在网站内完成；
- 不把评论跳转到 GitHub。

其他三个产品使用真实仓库能力描述，不编造客户数据、交易额、访问量或线上地址。

## 可访问性与性能

- 所有主题按钮、项目卡片和仓库链接可通过键盘操作，并有清晰焦点样式；
- 文本与背景对比度至少满足 WCAG AA；
- Canvas 使用 `aria-hidden="true"`，不承担信息表达；
- 页面无 JavaScript 时仍能读取内容和打开项目链接；
- 首屏背景不加载位图，项目截图继续延迟加载；
- Lighthouse 性能目标不低于 85，可访问性目标不低于 95；
- 浏览器控制台不得出现 error 或未处理 promise rejection。

## 非目标

- 不开发登录、主题云同步或用户自定义配色；
- 不把工具类仓库继续显示在博客项目目录；
- 不新增 WebGL、Three.js 或第三方粒子运行库；
- 不删除已经公开的独立工具 GitHub 仓库。

## 验收标准

- 三种主题可切换、刷新后记忆且视觉明显不同；
- 鼠标移动、点击、卡片悬停和滚动均有符合当前主题的反馈；
- 减少动态效果和移动端降级有效；
- 生产项目目录只有四个网站产品；
- 三个多仓库产品能看到完整关联源码；
- 个人博客明确显示永久免费、零成本部署、完全开源；
- 字体真实加载，页面不再声明未提供的 Geist；
- 单测、Astro 检查、生产构建、烟雾测试和浏览器点测全部通过。
