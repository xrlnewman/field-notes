# Field Notes 个人博客设计规格

## 1. 目标

建设一个无需服务器、数据库和付费域名即可长期运行的个人博客，集中展示个人项目、技术文章和公开交流。网站使用免费 `pages.dev` 子域名，内容与代码存放在公开 GitHub 仓库，静态页面托管在 Cloudflare Pages，评论与留言存放在 GitHub Discussions。

首版必须包含：

- 首页与个人简介
- 项目列表、项目详情、项目评论
- 文章列表、文章详情、标签筛选、文章评论及回复
- 全局留言板
- 中文全文搜索
- RSS、站点地图、SEO 元信息
- 深色/浅色主题与移动端适配
- 免费部署说明与内容维护说明

首版明确不包含：自建后台、匿名评论、数据库、用户账户、付费功能、邮件订阅和站内视频托管。

## 2. 品牌与视觉方向

项目代号和默认站点名为 **Field Notes**，默认字标为 `Y / Field Notes`。真实姓名、头像、社交链接和站点介绍统一放在站点配置文件中，修改配置即可替换，不散落在页面组件里。

视觉采用用户已确认的 **Editorial Green**：

- Mintlify 的阅读密度、代码展示和内容层级
- Vercel 的项目网格、极简导航和开发者气质
- 暖白背景、深墨绿正文、薄荷绿强调色
- 默认浅色，支持跟随系统和手动切换深色模式
- 不使用 Notion、Mintlify 或 Vercel 的商标、专有字体和完整品牌外观

### 2.1 设计令牌

| 类别 | 浅色 | 深色 | 用途 |
|---|---|---|---|
| 页面背景 | `#F8FAF7` | `#08110E` | 全局画布 |
| 卡片背景 | `#FFFFFF` | `#0E1B17` | 项目卡片、信息面板 |
| 主文字 | `#10201A` | `#EAF7F1` | 标题、正文 |
| 次文字 | `#66756E` | `#9DB1A8` | 摘要、元信息 |
| 强调色 | `#0B9873` | `#42D3A7` | 链接、标签、焦点状态 |
| 深色强调面 | `#10201A` | `#DDF8EE` | 精选项目、主要按钮 |
| 边框 | `#DCE8E2` | `#243B33` | 卡片和分隔线 |

- 中文正文使用系统无衬线字体栈；英文标题优先 Geist；代码优先 Geist Mono。
- 正文桌面端 17px/1.8，移动端 16px/1.75；文章正文最大宽度 760px。
- 圆角以 10px、14px、18px 三档为主；阴影克制，边框优先。
- 动效控制在 160–220ms，并尊重 `prefers-reduced-motion`。
- 所有可交互元素键盘可达，焦点轮廓使用强调色，正文对比度满足 WCAG AA。

## 3. 信息架构

```text
/
├── /projects
│   └── /projects/[id]
├── /articles
│   └── /articles/[id]
├── /tags/[tag]
├── /search
├── /guestbook
├── /about
├── /rss.xml
├── /sitemap-index.xml
└── /404
```

全局导航固定为：首页、项目、文章、留言、关于；右侧提供搜索入口、GitHub 链接和主题切换。移动端折叠成抽屉菜单，搜索与主题切换仍保持一跳可达。

## 4. 页面设计

### 4.1 首页

首页首屏为左对齐个人介绍，不使用大幅装饰图片。主标题为“做有用的东西，写真实的过程”，下面展示一句定位、GitHub/联系入口和当前状态。首屏右侧或下方展示精选项目卡。

后续顺序：精选项目（最多 3 个）→ 最新文章（最多 5 篇）→ 当前关注方向 → 简短个人介绍 → 留言板入口。首页只做内容入口，不复制完整列表页。

### 4.2 项目页

项目列表使用响应式双列网格，精选项目可跨两列。卡片展示名称、摘要、状态、年份、技术标签和演示/源码入口，不以无意义封面图占据主要空间。

项目详情依次展示：项目目标、问题背景、解决方案、关键功能、技术栈、结果/复盘、相关链接、评论区。不存在演示或源码链接时不渲染对应按钮。

### 4.3 文章页

文章列表提供标签筛选和搜索入口。文章卡以标题、摘要、发布日期、阅读时长和标签为主，不使用营销式大图瀑布流。

详情页桌面端使用“正文 + 右侧目录”结构；目录在窄屏隐藏。顶部展示标题、摘要、日期、更新时间、阅读时长和标签。正文支持标题锚点、代码高亮、复制按钮、引用、表格、图片说明和上一篇/下一篇。评论区位于正文和相关推荐之后。

### 4.4 留言板

留言页包含一段交流说明和固定 Giscus 讨论区。留言使用 `specific` 映射并绑定固定 term `global-guestbook`，确保路由或标题改变时仍指向同一讨论。

### 4.5 搜索页

使用 Pagefind 构建静态索引，前端按需加载搜索资源。搜索覆盖文章、项目和关于页；结果展示类型、标题、摘要和标签。空关键词显示搜索提示，无结果时提供文章和项目入口。

## 5. 技术架构

```text
Markdown / MDX 内容
        │ Astro Content Collections + Zod 校验
        ▼
Astro 静态构建 ── Pagefind 索引 ── RSS / Sitemap
        │
        ▼
Cloudflare Pages 静态资源
        │
        └── Giscus iframe ── GitHub Discussions
```

- 框架：当前稳定版 Astro，严格 TypeScript。
- 输出模式：纯静态，不安装 Cloudflare SSR 适配器。
- 内容：Astro Content Collections，使用 `src/content.config.ts`、glob loader 和 Zod schema。
- 样式：原生 CSS 变量和组件级样式，不引入大型 UI 框架。
- 搜索：Pagefind 扩展构建，支持中文索引。
- 评论：Giscus；文章和项目使用 `pathname`，全局留言使用固定 `specific` term。
- 部署：GitHub 公共仓库连接 Cloudflare Pages；构建命令 `npm run build`，输出目录 `dist`。

## 6. 模块边界

- `siteConfig`：唯一站点身份配置，包含名称、描述、作者、社交链接和基础 URL。
- `content.config`：文章和项目的数据契约；字段错误在构建阶段明确失败。
- `BaseLayout`：HTML、SEO、主题初始化、导航和页脚。
- `ArticleLayout`：文章头部、正文、目录、上下篇和评论组合。
- `ProjectLayout`：项目元信息、正文、链接和评论组合。
- `GiscusComments`：只负责 Giscus 配置、主题同步和缺失配置提示。
- `Search`：只负责加载 Pagefind、查询、状态和结果渲染。
- 卡片与标签组件只接收展示数据，不直接查询内容集合。

组件之间通过显式 props 和站点配置交互，页面负责查询/排序内容，展示组件不包含业务查询逻辑。

## 7. 内容模型

文章字段：`title`、`description`、`publishedAt`、可选 `updatedAt`、`tags`、可选 `cover`、`draft`、可选 `featured`。

项目字段：`title`、`description`、`publishedAt`、可选 `updatedAt`、`status`、`tech`、可选 `demoUrl`、可选 `repoUrl`、`featured`、`draft`。

生产构建过滤 `draft: true`；草稿不得依赖“隐藏”保护隐私，因为公开 GitHub 仓库仍能看到已提交文件。真正的私密草稿只能保留在本地且不提交。

## 8. 评论配置与降级

Giscus 使用以下公开环境变量：

- `PUBLIC_GISCUS_REPO`
- `PUBLIC_GISCUS_REPO_ID`
- `PUBLIC_GISCUS_CATEGORY`
- `PUBLIC_GISCUS_CATEGORY_ID`

配置完整时加载 Giscus。配置缺失时：开发环境显示清晰的配置提示；生产环境显示“评论区暂未启用”，页面其他内容继续正常工作。Giscus 加载失败不阻塞正文，且提供前往 GitHub Discussions 的备用入口（配置了仓库地址时）。

主题切换时向 Giscus iframe 发送主题更新消息，不重新加载整页。

## 9. 错误处理与边界状态

- 内容字段缺失或 URL 非法：构建失败并显示 schema 错误，不静默跳过。
- 不存在的文章、项目或标签：生成/显示 404 页面。
- 搜索资源在开发模式尚未生成：显示“请运行生产构建预览搜索”的说明。
- Pagefind 加载失败：显示重试按钮及文章/项目直接入口。
- 图片缺失：使用纯色信息卡，不使用破损图片图标。
- JavaScript 禁用：导航、内容、标签页和外链仍可使用；仅搜索、主题手动切换和评论不可用。
- 外链统一使用安全的 `rel="noopener noreferrer"`。

## 10. 性能与隐私

- 首屏不加载 Giscus；评论区进入视口附近后再加载。
- 无广告、无行为追踪、无第三方统计脚本。
- 头像和内容图片本地托管并压缩，单个资源控制在 1 MiB 内。
- 不托管视频文件，仅使用外部链接或按需嵌入。
- 字体优先本地或系统字体，避免依赖远程字体服务。

## 11. 测试与验收

自动验证：

- `npm run check`：Astro/TypeScript/内容 schema 校验通过。
- `npm test`：工具函数、内容排序、阅读时长和配置校验单测通过。
- `npm run build`：静态页面、RSS、站点地图和 Pagefind 索引生成成功。
- 构建产物检查：首页、项目、文章、标签、搜索、留言、关于、404均存在。

浏览器验收：

- 桌面和移动端导航、主题切换、搜索、标签筛选正常。
- 文章目录、代码复制、上一篇/下一篇正常。
- 项目和文章评论脚本得到正确映射；留言板使用固定 term。
- 页面无控制台 error/warning，无横向滚动。
- 键盘可以完成导航、搜索和主题切换。
- Lighthouse 的 Performance、Accessibility、Best Practices、SEO 目标均不低于 90。

部署验收：

- GitHub 公共仓库连接 Cloudflare Pages 后可用 `npm run build` 和 `dist` 完成部署。
- `pages.dev` 地址启用 HTTPS。
- 未配置任何付费资源、数据库或必须绑定银行卡的服务。

## 12. 免费性与迁移

站点运行时只依赖静态托管和 GitHub Discussions。所有正文、项目数据、样式和构建代码均在 Git 仓库中，可迁移到 GitHub Pages 或其他静态托管平台。评论数据保留在 GitHub Discussions。

免费服务政策可能调整，因此不承诺第三方“永久不变”；本方案通过纯静态输出、无数据库和可移植内容，将迁移成本控制到最低。
