# GitHub 个人资料与博客范围修正设计

## 背景

上一轮把 GitHub 个人资料需求扩展成了博客内的 `ProfileIdentity` 个人档案组件，导致首页右侧和关于页出现用户未要求的头像与档案卡片。本次修正以最新反馈为准：个人头像、昵称和简介只属于 GitHub；博客恢复到引入个人档案组件之前的版式。

## 目标

1. 首页右侧精确恢复上一版的三项数据：`7 年经验`、`4 个网站产品`、`永久免费`。
2. 关于页移除后来加入的头像档案卡片，并恢复原有标题字号和间距。
3. 删除博客专用的个人档案组件、头像文件和只为该组件增加的配置字段。
4. 保留博客已有的项目分类、项目截图、三种星空主题、站内 Giscus 评论留言和永久免费说明。
5. GitHub 保留昵称“许汝林”、现有简介和博客链接，只替换头像。

## 头像方向

GitHub 头像使用深蓝与冰蓝配色的极简专业标识，采用抽象 `XR` 字母结构和轻微空间层次。头像不使用像素风，不编造真人长相，不包含背景场景、口号、水印或细小装饰；主体必须在圆形裁剪和 32px 小尺寸下仍然清晰。

## 实现边界

- 博客回退只触及个人档案相关文件，不回退后续已经验收的项目截图。
- 不修改 GitHub 公开仓库的可见性、置顶项目、昵称、简介和博客链接。
- 新头像仅用于 GitHub，不重新放回博客页面。
- 不新增依赖，不重构无关页面。

## 验收标准

- 首页源码不再导入或渲染 `ProfileIdentity`，右侧出现三项上一版数据。
- 关于页不再渲染头像档案卡片。
- `ProfileIdentity.astro`、博客头像文件和对应配置字段被移除。
- 全量 `npm run verify` 通过。
- 1440×900 与 390×844 线上页面无横向溢出、字体加载完成、控制台无错误。
- GitHub 头像更新后公开资料仍显示昵称“许汝林”、既有简介与博客链接。

## 回滚策略

博客修改通过独立分支和 PR 合并；如线上异常可回滚该 PR。GitHub 头像保留本地最终 PNG，可再次上传或替换，不影响仓库代码。
