# HomeFlow 色彩系统设计

## 目标

把 HomeFlow 从“单一绿色模板”调整为有层级、有语义、可扩展的服务产品视觉系统。小程序、后台和博客展示截图使用同一套色彩语义，避免页面背景、按钮、价格和状态全部使用绿色。

## 设计原则

1. 绿色只表达信任、成功、已完成和可用状态，不作为全局默认色。
2. 关键行动使用暖橙色，强调预约、确认和价格信息。
3. 深青蓝负责品牌识别、标题、导航和高对比区域，建立专业与可靠感。
4. 暖白和中性灰负责画布、卡片、边框和次要文字，保证内容呼吸感。
5. 颜色对比度优先：正文与背景达到 WCAG AA，对禁用状态同时依赖文字和透明度，不只依赖颜色。

## 色彩令牌

| 令牌 | 色值 | 用途 |
| --- | --- | --- |
| `--ink` | `#183B43` | 标题、正文、导航图标 |
| `--ink-soft` | `#42626A` | 次级标题、说明文字 |
| `--muted` | `#718087` | 辅助文字、时间、占位信息 |
| `--brand` | `#183B43` | 品牌主色、选中导航、主视觉深色区 |
| `--brand-soft` | `#E7F0F0` | 主色浅底、标签和选中背景 |
| `--action` | `#F08A5D` | 主行动、预约按钮、价格和促销 |
| `--action-dark` | `#C9653E` | 橙色按下态和深色文字 |
| `--success` | `#3C9B79` | 成功、已完成、可用状态 |
| `--success-soft` | `#E6F5EF` | 成功浅底、成功标签 |
| `--warning` | `#D49A3A` | 余量提醒、待处理 |
| `--danger` | `#C85C5C` | 错误、取消、风险 |
| `--canvas` | `#F7F8F5` | 页面背景 |
| `--card` | `#FFFFFF` | 卡片和表单容器 |
| `--line` | `#E3E9E8` | 分割线和边框 |

## 使用比例

- 60%：`--canvas`、`--card` 和灰阶留白。
- 30%：`--ink`、`--brand` 和深色导航/英雄区域。
- 10%：`--action`、`--success`、`--warning`、`--danger` 等语义色。

## 页面映射

- 小程序首页：背景使用暖白；Hero 使用深青蓝；“立即预约”使用暖橙；价格使用暖橙；热门服务图标按服务类型使用薄荷绿、浅蓝、浅杏，不再统一绿色。
- 预约页：选中时间使用深青蓝边框 + 浅主色底；提交按钮使用暖橙；成功提示使用绿色。
- 订单页：订单状态使用语义色，状态栏不再用单色绿色；已完成绿色、待服务橙色、已取消红色。
- 师傅工作台：主操作使用暖橙，履约完成使用绿色，待处理使用琥珀色。
- 管理后台：导航和指标标题使用深青蓝；经营指标图表使用蓝、橙、绿、紫四组系列色，保证相邻数据可区分。
- 博客截图：截图中的色彩必须与源码令牌一致，设备框只负责展示，不额外叠加绿色遮罩。

## 不在本次范围

- 不改变业务流程、接口、数据结构或路由。
- 不更换字体、图标库和插画风格。
- 不把邻里社区项目复刻为下一款产品；下一款产品另建独立设计与实现计划。

## 验收标准

1. 小程序和后台源码中不再把绿色同时用于背景、主按钮、价格和所有状态。
2. 预约、成功、取消、待处理四类状态可通过颜色与文字共同识别。
3. 关键文字与背景的对比度满足 AA；禁用态不低于 3:1 的非文本对比要求。
4. 博客 HomeFlow 截图至少包含深青蓝、暖橙、灰阶三种层级，且与源码令牌一致。
5. 现有测试、构建和部署流程保持通过。
