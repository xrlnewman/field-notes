# EduFlow 教培课程与学员管理平台设计

## 目标

构建一套免费开源、可自部署的教培运营产品，拆成学员/家长端小程序仓库 `eduflow-miniapp` 与 Vue 管理后台 + Go Gin API 仓库 `eduflow-admin`，并将真实可运行页面接入许汝林个人博客。

## 选型依据

教育学习类小程序仍是高频使用场景，公开行业资料将课程报名、学员管理、教学互动和学习进度列为教培小程序的核心闭环。EduFlow 选择中小机构最容易验证价值的“课程—排课—报名—考勤—进度”链路，不复制复杂直播、支付分账或题库平台。

## 产品边界

### 小程序端

- 首页：今日课程、学习进度、待完成任务和快捷入口。
- 课程：按分类浏览课程，查看老师、课时、开课时间和剩余名额。
- 报名：提交报名后生成待确认记录，重复提交使用幂等键保护。
- 课表：查看已报名课程的日期、教室、老师和签到状态。
- 学习：查看课程进度、课后任务和老师反馈；所有演示数据均为虚构。
- 我的：展示学员身份、已报名课程、免费开源说明和 API 状态。

### 管理后台与 API

- 经营总览：报名人数、活跃学员、今日课程、待处理考勤和课程完成率。
- 课程中心：课程、分类、老师、课时和名额的只读/演示管理视图。
- 排课管理：按日期查看班级、教室和老师安排。
- 学员管理：查看学员、报名课程、学习进度和最近签到。
- 考勤处理：确认到课、请假和缺勤，写操作需要幂等键。
- 学习任务：查看任务提交状态与老师反馈入口。
- API：提供统一响应 `{ code, message, data }`，在未配置数据库时使用内存演示数据。

不包含真实支付、直播、未成年人敏感画像、医疗类数据、短信群发和商业 SaaS 计费，避免把博客演示误解为真实运营系统。

## 仓库关系

| 仓库 | 职责 | 公开入口 |
| --- | --- | --- |
| `eduflow-miniapp` | uni-app 学员/家长端，可构建 H5 与微信小程序 | `https://github.com/xrlnewman/eduflow-miniapp` |
| `eduflow-admin` | Vue 3 后台、Go Gin API、MySQL 8.4、Redis 8、Docker Compose | `https://github.com/xrlnewman/eduflow-admin` |

小程序只依赖 `/api/v1` 契约；API 未配置或网络失败时显示带“演示数据”标识的本地快照。博客只保存项目描述、截图与仓库链接，不复制两个仓库的业务源码。

## 视觉系统

- 主色：`#6D4AFF`，用于导航、选中状态和学习进度。
- 动作色：`#FF7A59`，只用于报名、确认和下一步操作。
- 完成色：`#18A889`，用于已签到、已完成和正常状态。
- 提醒色：`#E7A72B`，用于待处理、临近开课和容量提醒。
- 画布：`#F7F8FC`；墨色：`#1D2230`；边线：`#E7E9F1`。

视觉基调是“学习空间 + 运营台”：小程序使用柔和的紫色渐变、卡片式课表和进度环；后台使用深墨侧栏、浅画布和紫色强调，避免 StockFlow 的蓝橙复用。截图必须来自真实运行页面，统一保存为 PNG 并记录准确尺寸。

## 数据与错误处理

- MySQL 8.4 保存课程、班级、老师、学员、报名、考勤和任务状态。
- Redis 8 预留报名幂等键、名额短锁和热点课程缓存。
- 跨 IO 方法首参传递 `context.Context`；API 入站校验失败返回明确错误，不吞异常。
- 演示模式不写真实数据库，所有金额、姓名、课程数据使用虚构样例。
- 写操作使用业务唯一键 + `Idempotency-Key`，重复确认返回同一结果。

## 验收标准

1. 两个 GitHub 仓库为 public，README 显示本地启动、API 地址、免费开源和演示数据边界。
2. 小程序可以在无 API 时打开首页、课程、课表、学习和我的页面；测试覆盖 API 回退和幂等键。
3. 后台可切换总览、课程、排课、学员、考勤和任务；Go API 的健康检查、登录和核心 GET/POST 路由有测试。
4. `npm test`、H5 build、`go vet ./...`、`go test ./...`、后台 Web test/build 全部通过。
5. 博客新增“教育培训”分类和 EduFlow 详情页，至少 4 张真实 PNG 截图、两个仓库节点和站内 GitHub 登录评论。
6. 线上博客项目页、全部截图和两个 GitHub 仓库匿名访问返回 200；Linli 等已有项目不变。
