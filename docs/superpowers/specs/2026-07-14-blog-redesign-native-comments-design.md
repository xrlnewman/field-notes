# 个人博客作品集重构与原生评论设计

## 背景

当前站点的内容结构完整，但首页留白过多、标题尺度过大，项目截图和项目信息没有成为视觉中心。项目页虽然存在筛选代码，线上只有一个公开分类，访客难以感知完整的项目体系。评论使用 Giscus，界面文案和站外入口让访客误以为必须离开网站前往 GitHub 才能留言。

## 目标

- 把首页和项目页改成以真实项目截图为主角的“产品目录 / 创作者工作室”。
- 在首页和项目页明确展示项目分类、数量和筛选状态。
- 文章评论、项目评论和全局留言都在本站直接完成，支持一级回复。
- 不要求 GitHub 账号，不跳转到 GitHub，不增加付费服务。
- 保持静态内容、全文搜索、深色模式、RSS 和移动端体验。

## 非目标

- 本期不建设个人账号系统、邮件通知、点赞或富文本编辑器。
- 本期不提供后台管理页面；垃圾内容可通过 D1 命令删除，后续再增加管理端。
- 不为凑数量发布没有源码和真实说明的项目。

## 视觉方向

采用“产品目录 / 创作者工作室”方向，参考用户提供的商品项目网格，但不复制电商价格、销量等无关信息。

### 首页

- 首屏改为更紧凑的双栏结构：左侧一句价值主张和行动入口，右侧展示项目数、技术方向和持续交付年限。
- 首屏下方直接出现分类导航，分类使用独立卡片并显示公开项目数量。
- 项目卡片以 16:10 截图为主，分类和状态使用醒目标签；标题下只保留成果导向的一句话。
- “查看详情”是主操作，“源码”是次操作；两者都保持清晰且不互相抢夺注意力。
- 文章区降低视觉权重，继续放在项目之后。

### 项目页

- 页面标题使用一级标题，首屏同时显示公开项目总数和分类说明。
- 固定展示全部分类及数量，允许筛选到暂时没有公开项目的分类，并给出明确空状态。
- URL 查询参数保存分类选择，首页分类入口可直接落到对应筛选结果。
- 桌面端三列、平板两列、手机单列；按钮和筛选项满足 44px 触控高度。

### 分类体系

固定分类为：

1. 网站产品
2. 业务系统
3. 开发工具
4. 数据与搜索
5. AI 自动化

分类是长期作品体系，不因某一分类暂时为零而隐藏。

## 原生评论架构

Cloudflare Pages Functions 提供 `/api/comments`，Cloudflare D1 保存评论。页面仍由 Astro 静态生成，评论组件在浏览器中按资源加载。

### 资源标识

- 文章：`article:<slug>`
- 项目：`project:<slug>`
- 全局留言：`guestbook:global`

服务端只接受上述三种格式，不允许任意表名、路径或 SQL 片段进入查询。

### 数据模型

`comments` 表包含：`id`、`resource_type`、`resource_id`、`parent_id`、`author_name`、`content`、`created_at`、`status`、`ip_hash`。`parent_id` 只允许指向同一资源的顶级评论，因此回复最多一层。

### API

- `GET /api/comments?resource=...`：返回已发布的顶级评论及其回复，按创建时间正序排列。
- `POST /api/comments`：接收 `resource`、`authorName`、`content`、可选 `parentId` 和蜜罐字段 `website`。
- 成功响应使用 `{ "ok": true, "data": ... }`；校验失败使用 `{ "ok": false, "error": { "code", "message" } }`。

### 安全与失败处理

- 昵称 2–24 个字符，正文 2–1000 个字符；服务端重新裁剪并校验。
- 蜜罐字段非空时不写入数据库，但返回普通成功响应，避免向机器人暴露规则。
- 使用 `CF-Connecting-IP` 与部署密钥生成不可逆哈希；同一哈希 45 秒内只允许发布一次。
- 数据库异常返回 503，前端保留已填写内容并显示可重试提示，不吞错。
- 所有评论按纯文本渲染，不使用 `innerHTML`。

## 评论界面

- 评论列表、发布表单和回复表单全部出现在当前页面。
- 未登录即可使用昵称发布；明确提示留言公开展示。
- 发布成功后就地插入新评论，不刷新、不跳页。
- 没有评论时展示友好的首条留言提示；网络失败时提供重试按钮。
- 删除“前往 GitHub Discussions”入口和 GitHub 登录文案。

## 部署

- 新增 D1 数据库 `field-notes-comments` 和迁移脚本。
- Pages 项目绑定 `COMMENTS_DB`，密钥 `COMMENT_HASH_SECRET` 只写入 Cloudflare 环境，不入库、不输出。
- Astro 继续产出静态站点；Pages Functions 与 `dist` 一并部署。

## 验收标准

- 首页首屏在 1440px 和 390px 宽度下无横向溢出，项目截图在首屏后第一内容区可见。
- 首页能看到五个分类及数量；项目页五个分类都可筛选，查询参数可恢复状态。
- 文章、项目详情和留言板都显示本站评论表单，不含 GitHub Discussions 外链。
- 访客可以发布顶级评论和一级回复；刷新后内容仍存在。
- 非法资源、过短内容和 45 秒内重复发布得到明确响应。
- 单元测试、Astro 检查、静态构建测试和浏览器控制台检查全部通过。
