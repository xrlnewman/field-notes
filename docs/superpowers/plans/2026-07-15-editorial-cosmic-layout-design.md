# 编辑型星空作品集布局设计

## 背景

当前站点已经具备三套星空主题、项目分类、项目详情、GitHub 登录评论与留言，但布局比例偏向大字号海报：桌面首页首屏高度约 760px，项目详情标题上限约 102px，留言标题上限约 112px；项目卡片又存在横向精选卡与纵向普通卡两套高度，导致内容节奏断裂，项目截图和仓库关系不够突出。

本次只优化布局系统，不改内容模型、项目数量、三主题逻辑、星空交互和 Giscus 评论方式。

## 目标

- 让访客在首屏同时看清“我是谁、能做什么、项目入口”。
- 让四个网站项目成为首页和项目目录的主视觉，而不是被大标题与空白挤到下方。
- 让项目详情以“产品说明 + 仓库组成 + 源码入口”为清晰阅读路径。
- 让留言区像站内功能，而不是挂在大海报标题下的第三方嵌入。
- 三套主题共享完全一致的布局，只切换颜色、背景和氛围。

## 采用方向

采用“编辑型星空作品集”：保留暗色星空和细微动效，用更克制的字号、更明确的 12 列关系、更一致的卡片比例和类似技术杂志的细线分隔建立层级。

没有采用更电影化的方案，因为它会继续增加首屏空白；没有采用后台看板式方案，因为它会削弱个人品牌和作品叙事。

## 布局系统

### 全局

- 内容最大宽度保持 1200px，桌面左右安全边距 40px，手机安全边距 28px。
- 新增紧凑页面宽度变量 `--content-narrow: 960px`，用于留言和说明型页面。
- 标题采用 `text-wrap: balance`，正文采用 `text-wrap: pretty`，不更换现有 Inter Variable + Noto Sans SC Variable 字体。
- 桌面主标题上限收敛到 64px；项目详情上限 72px；留言页上限 64px；手机主标题上限 44px。
- 区块纵向间距以 24/40/64/88px 为主，避免 100px 以上的连续空段。

### 顶部导航

- 高度从 72px 收敛到 68px。
- 品牌、主导航、主题切换保持三段式，但降低主题切换的视觉重量。
- 手机仍使用现有菜单按钮；三主题使用短标签，触控区域不小于 44px。

### 首页

- 首屏最大高度从 760px 收敛到约 620px，桌面使用 7/5 两栏。
- 左侧为身份、主标题、简介和两个行动按钮；右侧把 7 年经验、4 个产品、永久免费整理成一张统一的能力面板。
- 免费开源说明并入首屏下方的横向信息条，减少独立卡片感。
- 项目区紧接首屏，四个项目全部使用一致的双列卡片，不再让个人博客横跨整行。
- 项目卡截图固定 16:10，正文区最小高度收敛到 280px；按钮放在底部统一基线。
- 文章区保持两列，但缩短卡片密度并弱化其视觉权重。

### 项目目录

- 顶部改成紧凑的“标题 + 项目数量 + 说明”横向信息区，标题不再占据大面积首屏。
- 分类筛选紧贴项目网格，桌面保持横向，手机可横向滚动。
- 项目卡与首页完全同构，确保浏览路径一致。

### 项目详情

- 首屏使用 8/4 两栏：左侧标题、简介和按钮，右侧为发布日期、状态、技术栈。
- 项目截图紧跟首屏，不再被大段元信息和空白推迟。
- 产品全景改为标题说明 + 仓库卡网格；仓库卡最小高度收敛，单仓项目也有明确源码入口。
- 正文保持约 760px 阅读宽度，项目索引从大圆点降为细小的章节标记。
- 评论区宽度与正文一致，保留 GitHub 登录后站内发布和回复。

### 留言页

- 页面最大宽度采用 960px，标题上限 64px。
- 标题、介绍和三条规则组成左侧说明；Giscus 评论区在右侧或下方形成完整功能面板。
- 桌面两栏，窄屏单栏；规则改为紧凑列表，不再制造 80px 的断层。

## 响应式与无障碍

- 1200px 以上：双列项目卡、详情 8/4、留言 4/8。
- 760px 以下：所有主布局单列，项目卡单列，标题最大 44px，按钮可换行。
- 390px 宽度不得出现横向滚动；所有按钮和筛选项最小触控高度 44px。
- 保持现有跳转正文、焦点环、语义标题、筛选 `aria-pressed` 与 Giscus 状态提示。
- `prefers-reduced-motion` 与粗指针设备继续关闭卡片倾斜和不必要动画。

## 验收标准

- 首页、项目列表、项目详情、留言页不再出现超过 72px 的主标题。
- 首页四个项目卡在桌面均为双列等宽，不再存在横跨整行的精选卡。
- 项目卡正文区最小高度不超过 290px，截图保持 16:10 且 `object-fit: contain`。
- 项目详情元信息进入右栏，留言页桌面为说明 + 评论区双栏。
- 三主题切换、项目分类、GitHub 登录评论与留言逻辑全部保持。
- `npm run verify` 通过，Astro 检查无错误或警告。
